# T-20260305-003: Add Clojure to ai-coding-lang-bench

## Overview

Add Clojure as a benchmark target language to the ai-coding-lang-bench repository.

## Changes

### 1. `benchmark.rb` - Add LANGUAGES entry

- Key: `'clojure'`
- Extensions: `clj`, `edn`
- Version command: `clojure --version`
- No extra_prompt needed (standard library only, no special type-checking)

### 2. `README.md` - Update language table

- Add Clojure to the Functional category as "Clojure (dynamic, JVM)"

### 3. `CLAUDE.md` - Update supported languages list

- Append `clojure` to the list

## Design Decisions

- **Category**: Clojure is a dynamic, functional language on the JVM. Placed in the Functional category alongside Scheme, OCaml, and Haskell.
- **JVM startup cost**: Not specially handled. The benchmark measures wall-clock time including startup, which is the fair comparison method used for all other languages (Java also runs on JVM).
- **Extensions**: `clj` for source files, `edn` for deps.edn configuration (used by Clojure CLI tools).
- **Version command**: `clojure --version` - standard way to check Clojure CLI version.
- **No extra_prompt**: Clojure uses only standard library (as required by the spec). No special type-checking tool integration needed.
